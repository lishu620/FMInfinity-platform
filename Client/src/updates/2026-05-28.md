# 更新说明

## 投票逻辑优化

### 流程简化

将原来的"文案选歌"环节改为"文案填写歌曲信息"，简化投票流程：

| 变更前 | 变更后 |
|---|---|
| 预处理 → 文案选歌（每人限选1首） → 投票（仅选中歌曲） | 预处理 → 文案填写歌曲信息（不限数量） → 投票（全部本期歌曲） |

### 具体变更

| # | 类型 | 说明 |
|---|---|---|
| 1 | ✨ | **MusicSelect 组件重写**：移除"每人限选1首"限制，文案组成员可点击任意歌曲卡片填写歌手、平台、链接等信息 |
| 2 | ✨ | **投票范围扩大**：`GET /vote/issue/:id/songs` 移除 `isSelected: true` 过滤，全部本期歌曲均可投票 |
| 3 | ✨ | **状态标签更新**：`IssueDetail`、`IssueList`、`SubmissionStatus` 中"文案选歌"→"文案填写歌曲信息"，"提交中"→"填写歌曲信息" |
| 4 | 🐛 | **修复文案编辑歌手显示"未知"**：`GET /issue/:id/review` 和 `GET /issue/:id/show` 增加 `Vsinger` 关联查询，前端改用 `song.vsingers` 显示歌手 |
| 5 | 🛠️ | **代码清理**：`issue.js` 顶部统一导入 `Vsinger`，移除路由内重复的局部 `require` |

### 影响范围

- `Client/src/components/MusicSelect.vue` — 重写
- `Client/src/views/IssueDetail.vue` — 状态标签
- `Client/src/views/IssueList.vue` — 状态文字
- `Client/src/views/SubmissionStatus.vue` — 步骤标题 + 状态文字
- `Client/src/views/IssueReview.vue` — 歌手显示修复 + 状态文字
- `Client/src/views/IssueShow.vue` — 歌手显示修复 + 状态文字
- `Server/routes/vote.js` — 移除 `isSelected` 过滤
- `Server/routes/issue.js` — `Vsinger` 关联查询 + 导入统一
