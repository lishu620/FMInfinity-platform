# 更新说明

## 功能更新

本次更新添加了**通知**模块，实现系统内消息推送功能。

- **稿件状态变更通知**：当稿件状态发生变化时（如草稿→收歌中→投票中→已确认→已发布），系统会自动向**所有注册用户**发送一条通知，告知变更详情。
- **统一通知发送接口**：提供了 `sendNotice` 通用函数，支持三种接收人模式：单人（single）、角色组（group）、全员（all），方便后续扩展其他业务通知（如投票提醒、文案认领等）。
- **通知中心**：用户可在前端“我的通知”页面查看所有未读/已读通知，支持单条标记已读和跳转至相关业务页面。

## 模块更新

本次更新添加了 **sendNotice** 通用通知模块（位于 `utils/noticeSend.js`），并基于它实现了**稿件状态变更全员通知**功能。

### sendNotice 模块使用说明

#### 函数签名

```javascript
async function sendNotice(params)
```

#### 参数说明

| 参数名       | 类型   | 必填 | 说明                                                                                                                  |
| ------------ | ------ | ---- | --------------------------------------------------------------------------------------------------------------------- |
| `type`       | string | 是   | 通知范围：`'single'`（单人）、`'group'`（按角色组）、`'all'`（全员）                                                  |
| `target`     | mixed  | 条件 | `type='single'` 时为用户 ID；`type='group'` 时为角色名称（需与 `Status.name` 匹配）；`type='all'` 时可不传或传 `null` |
| `title`      | string | 是   | 通知标题，例如 `"稿件状态变更通知"`                                                                                   |
| `content`    | string | 是   | 通知正文，支持任意文本                                                                                                |
| `bizType`    | string | 否   | 业务类型，用于前端区分不同业务，默认 `'system'`                                                                       |
| `bizId`      | any    | 否   | 关联的业务主键（如稿件 ID、评论 ID），默认 `null`                                                                     |
| `jumpPath`   | string | 否   | 点击通知后的前端跳转路径，如 `"/review"`，默认空字符串                                                                |
| `sendUserId` | number | 否   | 触发通知的操作人 ID，用于追溯，默认 `null`                                                                            |

#### 调用示例

**1. 发送给单个用户**

```javascript
await sendNotice({
  type: "single",
  target: 123,
  title: "您的稿件已通过审核",
  content: "恭喜，稿件《XXX》已发布",
  bizType: "review",
  bizId: 456,
  jumpPath: "/my-issues",
  sendUserId: 1,
});
```

**2. 发送给某个角色组**

```javascript
await sendNotice({
  type: "group",
  target: "文案组",
  title: "新稿件待确认",
  content: "请前往稿件查看页认领歌曲",
  bizType: "issue_status",
  bizId: issue.id,
  jumpPath: "/review",
  sendUserId: req.user.id,
});
```

**3. 发送给所有用户（全员广播）**

```javascript
await sendNotice({
  type: "all",
  title: "系统维护通知",
  content: "今晚 22:00 服务器将短暂重启，请提前保存工作",
  bizType: "announcement",
  jumpPath: "/notice",
});
```
